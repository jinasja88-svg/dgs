## 1. 프로젝트 개요

**딸깍소싱** - 1688(알리바바) 중국 소싱 대행 플랫폼
- 원클릭으로 중국 상품 소싱, 투명한 수수료, 실시간 환율 제공
- Next.js 웹앱 + Supabase 백엔드 + Streamlit 관리자 대시보드

## 5. 데이터베이스 스키마

### 핵심 테이블

**profiles** - 사용자 프로필 (auth.users INSERT 시 자동 생성)
- id, email, name, phone, avatar_url
- subscription_plan ('free'|'basic'|'pro'), subscription_expires_at

**products** - 상품 (강의/소프트웨어)
- type ('course'|'software'), title, slug, description, content_html
- price, discount_price, thumbnail_url, category_id, is_published

**categories** - 카테고리 (계층구조, parent_id)

**cart_items** - 장바구니 (user_id + product_id unique)

**wishlist_items** - 위시리스트 (user_id + product_id unique)

**orders / order_items** - 주문 및 주문 상세
- status: 'pending'|'paid'|'cancelled'|'refunded'

**reviews** - 리뷰 (user_id + product_id unique, rating 1-5)

**notices** - 공지사항 (is_pinned)

**subscription_payments** - 구독 결제 이력

### 소싱 테이블

**sourcing_products** - 1688 상품 캐시
- product_id (1688 ID), title, title_zh, price_cny, price_krw
- images (JSONB), skus (JSONB), seller (JSONB), stock

**sourcing_orders** - 소싱 주문
- status: 'pending'|'paid'|'purchasing'|'shipping'|'delivered'|'cancelled'
- items (JSONB), total_krw, service_fee, shipping_fee
- shipping_address (JSONB), tracking_number

**exchange_rates** - 환율 캐시
- from_currency, to_currency, rate, fetched_at

### RLS 정책
- profiles: 본인 읽기/쓰기
- products: 공개 읽기(published), 관리자 관리
- cart/wishlist: 본인 CRUD
- orders: 본인 읽기/쓰기
- reviews: 공개 읽기, 본인 쓰기
- notices: 공개 읽기, 관리자 관리
- sourcing_orders: 본인 읽기/쓰기, 관리자 수정

### 트리거
- auth.users INSERT → profiles 자동 생성
- products, sourcing_products, sourcing_orders → updated_at 자동 갱신

## 6. 인증 & 라우트 보호

**미들웨어 (`middleware.ts`):**
1. 세션 갱신 - Supabase auth 쿠키 유지
2. 보호 라우트 → 미인증 시 `/login` 리다이렉트: `/mypage`, `/checkout`, `/sourcing-orders`
3. 관리자 라우트 (`/admin`) → `ADMIN_EMAILS` 환경변수 체크
4. 인증 리다이렉트 → 로그인 상태에서 `/login` 방문 시 `/mypage`로 이동

**OAuth:** Google + Kakao (Supabase Auth)

## 12. 설계 원칙

1. **소싱이 핵심** - 모든 신규 기능은 소싱 중심
2. **개발용 목업 데이터** - 실제 API 연동은 TODO
3. **SSR 호환 Supabase** - `@supabase/ssr`로 서버/클라이언트 분리
4. **RLS로 보안** - DB 레벨에서 접근 제어
5. **JSONB 활용** - 동적 데이터(주문 아이템, 배송 주소)
6. **이메일 화이트리스트 관리자** - 환경변수 기반 간단 관리자 인증
7. **Tailwind + CSS 변수** - 커스텀 디자인 토큰
8. **Next.js App Router** - Pages Router 사용 안 함
9. **TypeScript strict** - 전체 타입 안전성
10. **모바일 퍼스트** - 반응형 디자인
